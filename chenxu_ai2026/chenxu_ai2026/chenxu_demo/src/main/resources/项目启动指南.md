### 3.2 Prometheus + Grafana 指标监控（推荐）

Prometheus 每 15 秒从两个服务的 `/actuator/prometheus` 端点拉取 JVM/HTTP/Kafka 指标，Grafana 负责可视化。

#### 下载

```bash
# ===== Prometheus（~100MB）=====
curl -L -o tools/prometheus-3.3.0.windows-amd64.zip \
  https://github.com/prometheus/prometheus/releases/download/v3.3.0/prometheus-3.3.0.windows-amd64.zip
unzip tools/prometheus-3.3.0.windows-amd64.zip -d tools/

# ===== Grafana（~120MB）=====
curl -L -o tools/grafana-11.6.0.windows-amd64.zip \
  https://dl.grafana.com/oss/release/grafana-11.6.0.windows-amd64.zip
unzip tools/grafana-11.6.0.windows-amd64.zip -d tools/
```

#### 启动

```bash
# 终端 1：Prometheus（配置文件已在 tools/prometheus.yml）
cd tools/prometheus-3.3.0.windows-amd64/
./prometheus.exe --config.file=../../tools/prometheus.yml

# 终端 2：Grafana
cd tools/grafana-v11.6.0/bin/
./grafana-server.exe
```

#### 配置 Grafana

1. 打开 `http://localhost:3000`（默认 `admin` / `admin`）
2. **添加数据源**：Configuration → Data Sources → Add → Prometheus
   - URL: `http://localhost:9090`
   - Save & Test
3. **导入仪表盘**：Dashboards → Import → 输入以下 ID：

| Dashboard ID | 名称 | 监控内容 |
|---|---|---|
| **19004** | Spring Boot 3.x Statistics | JVM 内存/GC/线程/HTTP |
| **18276** | Spring Cloud Gateway | 网关路由 QPS/延迟/状态码 |
| **18210** | Kafka Overview | Consumer Lag / Producer 吞吐 |

#### 验证

```bash
# Prometheus 指标端点（应先启动 chenxu_demo 和 chenxu-gateway）
curl -s http://localhost:8081/actuator/prometheus | head -5
curl -s http://localhost:8080/actuator/prometheus | head -5

# Prometheus 控制台
# 打开 http://localhost:9090 → Status → Targets → 确认两个 target 都是 UP
# Graph 页面查询: http_server_requests_seconds_count
```

## 八、常见问题

### Q1: Maven 编译报 "无效的目标发行版: 21"
```bash
# 确认 JAVA_HOME 指向 JDK 21
export JAVA_HOME="/c/Program Files/Java/jdk-21.0.10"
```

### Q2: 启动报 Nacos 连接超时
- 确认 Nacos Server 在 192.168.249.129:8848 正常运行
- 检查 namespace ID 是否匹配

### Q3: Kafka Topic 不存在
```bash
kafka-topics.sh --create --topic HASH_PAR_CX   --partitions 3 --bootstrap-server 192.168.249.129:9092
kafka-topics.sh --create --topic TracingTopic --partitions 3 --bootstrap-server 192.168.249.129:9092
```

### Q4: 前端 401 / 登录失败
- 确认 MySQL `sys_user` 表已创建
- 确认后端 AuthController 启动正常
- 浏览器 F12 → Application → Local Storage → 清空 token → 重新登录

### Q5: Redis 连接失败
- 确认 Redis 在 192.168.249.129:6379 运行
- 密码是否正确：`snCJadmin-1`

### Q6: 前端跨域报错
- 确认后端 `CorsConfig` 已加载（日志搜索 CorsConfig）
- 或确认 Vite proxy 配置生效（`vite.config.ts` 中配置了所有路径代理）

### Q7: Prometheus Target 显示 DOWN
- 确认 `chenxu_demo` 和 `chenxu-gateway` 已启动
- 确认 `/actuator/prometheus` 端点可达：`curl http://localhost:8081/actuator/prometheus`
- 检查 `tools/prometheus.yml` 中 targets 端口是否正确

### Q8: Grafana 仪表盘无数据
- 确认 Prometheus 数据源 URL 正确：`http://localhost:9090`
- 确认 Prometheus 中已有数据：打开 `http://localhost:9090` → Graph 查询任意指标
- 检查仪表盘的数据源下拉是否选择了 Prometheus

### Q9: `/actuator/prometheus` 返回 404
- 确认 `pom.xml` 中已添加 `micrometer-registry-prometheus` 依赖
- 确认 `application.yml` 中 `management.endpoints.web.exposure.include` 包含 `prometheus`
- 重启服务后重试

## 九、一键启动脚本（开发用）

```bash
#!/bin/bash
# 保存为 start-all.sh

export JAVA_HOME="/c/Program Files/Java/jdk-21.0.10"

echo "=== 1. 检查连通性 ==="
HOST=192.168.249.129
for port in 3306 8848 9092 6379; do
  nc -z -w2 $HOST $port && echo "  $HOST:$port ✓" || echo "  $HOST:$port ✗ 不通!"
done

echo ""
echo "=== 2. 启动后端 (chenxu_demo :8081) ==="
cd "$(dirname "$0")/chenxu_ai2026/chenxu_demo"
mvn spring-boot:run &
BACKEND_PID=$!
sleep 20

echo ""
echo "=== 3. 启动前端 (chenxu-vue :5173) ==="
cd "$(dirname "$0")/chenxu_ai2026/chenxu-vue"
npm run dev &
FRONTEND_PID=$!

echo ""
echo "=== 启动完成 ==="
echo "后端:  http://localhost:8081"
echo "前端:  http://localhost:5173"
echo ""
echo "Ctrl+C 停止所有服务"
trap "kill $BACKEND_PID $FRONTEND_PID 2>/dev/null; exit" INT TERM
wait
```
