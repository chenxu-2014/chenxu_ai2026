### 第一步：Nacos 配置中心

#### 作用
将数据库、Redis、Kafka 等业务配置从本地 `application.yml` 迁移到 Nacos，实现**配置集中管理**和**动态刷新**。

#### 依赖
```xml
<!-- chenxu_demo/pom.xml & chenxu-gateway/pom.xml -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
</dependency>
```

#### 配置（application.yml）
```yaml
spring:
  application:
    name: chenxu-demo            # 必须：作为 Nacos Data ID 前缀
  cloud:
    nacos:
      config:
        server-addr: 192.168.249.129:8848
        namespace: 589e3087-3772-45e7-95a0-6130a8a038f3   # 命名空间 ID
        group: DEFAULT_GROUP
        file-extension: yaml
  config:
    import:
      - optional:nacos:chenxu-demo.yaml   # 导入 Nacos 配置，optional=服务不可用时兜底
```

#### 关键注解

| 注解 | 作用 | 使用位置 |
|------|------|----------|
| `@RefreshScope` | 标记 Bean 支持**配置热刷新**，Nacos 配置变更后无需重启 | `KafkaTopicConfig.java:11` `KafkaMessageProducer.java:17` |

```java
// KafkaTopicConfig.java — @RefreshScope 用于配置热刷新
@Configuration
@RefreshScope                            // ← 此 Bean 的 @Value 值会在 Nacos 配置变更后自动刷新
public class KafkaTopicConfig {

    @Value("${kafka.topic}")
    private String topic;               // Nacos 上改 kafka.topic，这里自动更新

    @Value("${tracing.topic:TracingTopic}")
    private String tracingTopic;

    @Bean
    public NewTopic hashParTopic() {
        return TopicBuilder.name(topic).partitions(partitions).replicas(1).build();
    }

    @Bean
    public NewTopic tracingTopic() {
        return TopicBuilder.name(tracingTopic).partitions(3).replicas(1).build();
    }
}
```

**@RefreshScope 原理**：Spring Cloud 会为标注了 `@RefreshScope` 的 Bean 创建一个**懒加载代理**。当收到配置变更事件（RefreshScopeRefreshedEvent）时，销毁原 Bean 缓存，下次访问时重新创建，从而读到最新的 `@Value` 值。

#### Nacos 控制台操作
1. 进入 Nacos → 命名空间 → 新建 `shiyue`（复制其 UUID 作为 `namespace` 值）
2. 进入配置管理 → 选择 `shiyue` 命名空间 → 新建配置
   - **Data ID**: `chenxu-demo.yaml`
   - **Group**: `DEFAULT_GROUP`
   - **配置内容**: 将业务配置（datasource/redis/kafka/mybatis）写入

#### 引入前后对比
| | 引入前 | 引入后 |
|---|---|---|
| 配置存储 | 本地 application.yml | Nacos 集中管理 |
| 修改配置 | 改文件 → 重启 | 改 Nacos → 自动刷新（`@RefreshScope`） |
| 多环境 | 多个配置文件 | Nacos namespace 隔离 |

### 第二步：Nacos 注册中心

#### 作用
将服务注册到 Nacos，让 Gateway 和其他服务能**通过服务名发现和调用**，无需硬编码 IP:Port。

#### 依赖
```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
</dependency>
```

#### 配置（application.yml）
```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: 192.168.249.129:8848
        namespace: 589e3087-3772-45e7-95a0-6130a8a038f3
        group: DEFAULT_GROUP
```

#### 关键注解

| 注解 | 作用 | 使用位置 |
|------|------|----------|
| `@EnableDiscoveryClient` | 开启**服务注册与发现**，启动时自动向 Nacos 注册 | `cx_demo.java:8` `GatewayApplication.java:15` |

```java
// cx_demo.java — 主启动类
@SpringBootApplication
@EnableDiscoveryClient                  // ← 开启 Nacos 服务发现
public class cx_demo {
    public static void main(String[] args) {
        SpringApplication.run(cx_demo.class, args);
    }
}
```

**@EnableDiscoveryClient 原理**：触发 `DiscoveryClient` 自动配置，启动时向 Nacos 发送注册请求（服务名 + IP + 端口 + 健康状态），并启动心跳定时任务维持注册状态。同时拉取 Nacos 上的服务列表缓存到本地，供 `lb://serviceName` 调用。

#### 服务调用方式
```yaml
# Gateway 路由 — lb:// 协议自动从 Nacos 获取实例列表并负载均衡
spring:
  cloud:
    gateway:
      routes:
        - id: chenxu-demo-route
          uri: lb://chenxu-demo        # ← 服务名对应 spring.application.name
          predicates:
            - Path=/meter/**,/kafka/**,/redis/**
```

**`lb://` 工作原理**：
1. Gateway 收到请求 → 根据 Path 匹配到路由
2. `lb://chenxu-demo` 触发 `LoadBalancerClient` 从 Nacos 获取 `chenxu-demo` 的实例列表
3. 选择一个实例（轮询/加权），转发请求

#### Nacos 控制台验证
进入 Nacos → 服务管理 → 服务列表 → 选择 `shiyue` 命名空间 → 可看到 `chenxu-demo` 和 `chenxu-gateway`

### 第三步：Spring Cloud Gateway 网关

#### 作用
统一入口，实现**请求路由**、**JWT 鉴权**、**限流**、**日志处理**。

#### 依赖
```xml
<!-- chenxu-gateway/pom.xml -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-gateway</artifactId>       <!-- 网关核心 -->
</dependency>
<dependency>
    <groupId>io.jsonwebtoken</groupId>
    <artifactId>jjwt-api</artifactId>                           <!-- JWT -->
</dependency>
```

#### 架构说明
```
浏览器请求 → :8080 (Gateway)
                │
                ├─ SentinelGatewayFilter    (流控检查，优先级最高，由 SentinelSCGAutoConfiguration 自动注册)
                ├─ RateLimitFilter          (手动限流 Demo，已被 Sentinel 替代)
                ├─ JwtAuthFilter            (JWT 鉴权，白名单放行)
                ├─ RequestLogFilter         (请求日志 + 耗时)
                │
                └─ 路由转发 lb://chenxu-demo → :8081
```

| 过滤器 | 优先级 (Order) | 功能 |
|--------|---------------|------|
| SentinelGatewayFilter | `HIGHEST_PRECEDENCE` | 网关层流控，超限返回 429 |
| JwtAuthFilter | `-100` | 校验 `Authorization: Bearer <token>` |
| RequestLogFilter | `-50` | 记录 方法 + 路径 + IP + 状态码 + 耗时 |
| RateLimitFilter | `-80` | 手动限流 Demo（已被 Sentinel 替代） |

#### 关键注解

| 注解 | 作用 | 使用位置 |
|------|------|----------|
| `@GlobalFilter` | 标记为**全局过滤器**，对所有路由生效 | — |
| `@Component` | 让 Spring 管理过滤器实例 | `JwtAuthFilter.java:28` `RequestLogFilter.java:19` |
| `@Order(-100)` | 控制过滤器**执行顺序**，数值越小越先执行 | `JwtAuthFilter` `RequestLogFilter` |
| `@Configuration` | 标记 Gateway 规则配置类 | `SentinelGatewayConfig.java:28` |
| `@PostConstruct` | 启动时**初始化规则**（API 分组 + 流控阈值） | `SentinelGatewayConfig.java:38` |

```java
// JwtAuthFilter.java — JWT 鉴权 + 白名单
@Component
public class JwtAuthFilter implements GlobalFilter, Ordered {

    // 白名单路径 — 直接放行，不校验 Token
    private static final List<String> WHITELIST = List.of(
            "/auth/login", "/auth/register", "/public/",
            "/actuator/", "/meter/getkey/", "/meter/count",
            "/sentinel/", "/order/"
    );

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        String path = exchange.getRequest().getURI().getPath();
        if (isWhitelisted(path)) return chain.filter(exchange);          // 白名单放行

        String authHeader = exchange.getRequest().getHeaders()
                .getFirst(HttpHeaders.AUTHORIZATION);
        if (authHeader == null || !authHeader.startsWith("Bearer "))
            return unauthorized(exchange, "缺少 Authorization 头");       // 401

        Claims claims = JwtUtil.validate(authHeader.substring(7));
        if (claims == null)
            return unauthorized(exchange, "Token 无效或已过期");          // 401

        // 鉴权通过，将用户信息透传给下游
        ServerHttpRequest mutated = exchange.getRequest().mutate()
                .header("X-User-Id", JwtUtil.getUserId(claims)).build();
        return chain.filter(exchange.mutate().request(mutated).build());
    }

    @Override public int getOrder() { return -100; }   // 在 RequestLogFilter(-50) 之前执行
}
```

```java
// RequestLogFilter.java — 记录请求和响应
@Component
public class RequestLogFilter implements GlobalFilter, Ordered {

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        String method = request.getMethod().name();
        String path = request.getURI().getPath();
        long start = System.currentTimeMillis();

        log.info("[网关] → {} {} | 来源IP: {}", method, path, ip);     // 请求进入

        return chain.filter(exchange).then(Mono.fromRunnable(() -> {
            long elapsed = System.currentTimeMillis() - start;
            int status = exchange.getResponse().getStatusCode().value();
            log.info("[网关] ← {} {} | 状态: {} | 耗时: {}ms",        // 响应完成
                    method, path, status, elapsed);
        }));
    }

    @Override public int getOrder() { return -50; }
}
```

#### JWT 鉴权流程
```
请求 → 白名单？→ 放行
         │
         ↓ 否
    Authorization 头存在？→ 否 → 401
         │
         ↓ 是
    Token 有效？→ 否 → 401
         │
         ↓ 是
    解析出 userId → 写入 X-User-Id 头 → 转发给下游
```

### 第四步：Sentinel 熔断限流

#### 作用
**三层防护**：网关层限流 → 服务层流控 → 服务层熔断，防止流量洪峰、慢调用、异常率过高拖垮系统。

#### 依赖
```xml
<!-- chenxu_demo/pom.xml -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-sentinel</artifactId>
</dependency>

<!-- chenxu-gateway/pom.xml -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-sentinel</artifactId>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-alibaba-sentinel-gateway</artifactId>   <!-- 网关适配 -->
</dependency>
```

#### 配置（application.yml）
```yaml
spring:
  cloud:
    sentinel:
      transport:
        dashboard: 192.168.249.129:8858    # Sentinel Dashboard 控制台
        port: 8719                          # 客户端通信端口
      eager: true                           # 启动即初始化

# Gateway 额外配置 — 限流时返回统一 JSON 响应
spring.cloud.sentinel.scg.fallback:
  mode: response
  response-status: 429
  response-body: '{"code":429,"msg":"Sentinel 网关限流，请稍后重试"}'
```

#### 规则体系

```
┌──────────────────────────────────────────────────────┐
│                 Sentinel 防护全景                     │
├───────────────┬──────────────────────────────────────┤
│ 层级           │ 规则                                  │
├───────────────┼──────────────────────────────────────┤
│ Gateway 网关   │ meter_api:   1 QPS（全量加载，重操作）  │
│ （按 API 分組） │ kafka_api:  50 QPS                   │
│               │ redis_api: 200 QPS                    │
├───────────────┼──────────────────────────────────────┤
│ Demo 服务层    │ loadAllToRedis: 并发≤1 + 慢调用>50%熔断│
│               │ kafkaSend:     QPS≤100 + 异常>30%熔断 │
│               │ sentinelFlowTest: QPS=2（测试用）      │
│               │ sentinelDegradeTest: 异常>50%熔断      │
│               │ sentinelSlowTest: 慢调用>50%熔断       │
└───────────────┴──────────────────────────────────────┘
```

#### 规则类型

| 规则类型 | 说明 | 示例 |
|----------|------|------|
| **FlowRule（流控）** | 限制 QPS 或并发线程数 | `loadAllToRedis` 并发线程 ≤ 1 |
| **DegradeRule（熔断）** | 慢调用比例/异常比例超过阈值时熔断 | 异常率 > 50% 熔断 10s |

#### 关键注解

| 注解 | 作用 | 使用位置 |
|------|------|----------|
| `@SentinelResource` | 标记需要**流控/熔断保护**的方法 | `CmMeterServiceImpl.java:66` `KafkaMessageProducer.java:32` `SentinelTestController.java:33` |
| `@PostConstruct` | 应用启动时**初始化规则** | `SentinelRulesInit.java:41` `SentinelGatewayConfig.java:38` |

```java
// CmMeterServiceImpl.java — 重量级操作加 Sentinel 保护
@Service
public class CmMeterServiceImpl implements CmMeterService {

    @Override
    // blockHandler = 被流控/熔断时调用 | fallback = 业务异常时调用
    @SentinelResource(
            value = "loadAllToRedis",                          // 资源名，规则中引用
            blockHandlerClass = SentinelBlockHandler.class,    // 兜底类
            blockHandler = "loadAllToRedisBlock",              // 兜底方法（必须 static）
            fallbackClass = SentinelBlockHandler.class,
            fallback = "loadAllToRedisFallback"
    )
    public String loadAllToRedis() {
        // ... 1000w 数据全量加载到 Redis
    }
}
```

```java
// SentinelBlockHandler.java — 统一兜底处理器
public class SentinelBlockHandler {

    // blockHandler 方法签名 = 原参数 + BlockException
    public static String loadAllToRedisBlock(BlockException e) {
        if (e instanceof FlowException)
            return "{\"code\":429,\"msg\":\"全量加载任务已在执行中，请稍后重试\"}";
        if (e instanceof DegradeException)
            return "{\"code\":503,\"msg\":\"数据加载服务暂时不可用（熔断中）\"}";
        return "{\"code\":429,\"msg\":\"请求被 Sentinel 拦截\"}";
    }

    // fallback 方法签名 = 原参数 + Throwable
    public static String loadAllToRedisFallback(Throwable t) {
        return "{\"code\":500,\"msg\":\"数据加载失败: " + t.getMessage() + "\"}";
    }
}
```

```java
// SentinelRulesInit.java — 启动时初始化流控和熔断规则
@Component
public class SentinelRulesInit {

    // 资源名称常量（需与 @SentinelResource 的 value 一致）
    public static final String RESOURCE_LOAD_ALL = "loadAllToRedis";
    public static final String RESOURCE_KAFKA_SEND = "kafkaSend";

    @PostConstruct
    public void init() {
        initFlowRules();      // 流控规则
        initDegradeRules();   // 熔断规则
    }

    private void initFlowRules() {
        FlowRule loadRule = new FlowRule(RESOURCE_LOAD_ALL)
                .setGrade(RuleConstant.FLOW_GRADE_THREAD)   // 按并发线程数限流
                .setCount(1);                                 // 最多 1 个线程

        FlowRule kafkaRule = new FlowRule(RESOURCE_KAFKA_SEND)
                .setGrade(RuleConstant.FLOW_GRADE_QPS)       // 按 QPS 限流
                .setCount(100);                               // 每秒 100 次

        FlowRuleManager.loadRules(List.of(loadRule, kafkaRule));
    }

    private void initDegradeRules() {
        DegradeRule loadDegrade = new DegradeRule(RESOURCE_LOAD_ALL)
                .setGrade(CircuitBreakerStrategy.SLOW_REQUEST_RATIO.getType()) // 慢调用比例
                .setCount(200)                           // RT > 200ms 算慢调用
                .setTimeWindow(30)                       // 熔断 30 秒
                .setMinRequestAmount(5)                  // 最少 5 个请求
                .setSlowRatioThreshold(0.5);             // 慢调用 > 50% 触发

        DegradeRuleManager.loadRules(List.of(loadDegrade));
    }
}
```

#### 熔断限流触发流程
```
请求进入 → Sentinel Entry
              │
              ├─ FlowRule 检查 → QPS/线程数超限？→ 限流 → blockHandler → 返回 429
              │
              ├─ DegradeRule 检查 → 熔断器打开？→ 熔断 → blockHandler → 返回 503
              │
              └─ 通过 → 执行业务方法
                            │
                            ├─ 正常 → 记录 RT/成功数 → 返回
                            └─ 异常 → 记录异常 → fallback → 返回 500
```

### 第五步：分布式链路追踪

#### 作用

```xml
<dependency>

</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>      <!-- 自动配置 Tracer -->

```

#### 配置（application.yml）
```yaml

  tracing:
    sampling:
      probability: 1.0                        # 100% 采样（Demo 环境，生产建议 0.1）
  zipkin:
    tracing:

logging:
  pattern:
    level: "%5p [${spring.application.name:},%X{traceId:-},%X{spanId:-}]"
    # 日志示例：INFO [chenxu-demo,abc123,def456] 交易链路开始
```

#### 调用链透传原理
```
浏览器 → Gateway                          → chenxu-demo
         traceId: 6a2baed3...              traceId: 6a2baed3... (透传)
         spanId:  gateway-span             spanId:  demo-span   (新建)
         │                                  │
         │  HTTP Header:                     │
         │  b3: 6a2baed3...-gateway-span-1   │
         │  ───────────────────────────────► │
         │                                  │
         │                   ┌──────────────┤
         │                   │ 子 Span      │
         │                   │ ORDER    ────┤
         │                   │ INVENTORY ───┤
         │                   │ PAYMENT  ────┤
         │                   └──────────────┘
         │                                  │
         └────────── Zipkin Server ─────────┘
                    :9411 可视化
```

traceId 和 spanId 通过 HTTP Header `b3` 自动在服务间透传，无需任何代码处理。

#### 关键 API（brave.Tracer）

| API | 作用 | 使用位置 |
|-----|------|----------|
| `tracer.currentSpan()` | 获取当前 Span | `TransactionService.java` |
| `tracer.nextSpan().name("ORDER").start()` | **创建子 Span** | `TransactionService.java:createOrder()` |
| `tracer.withSpanInScope(span)` | 将 Span 设为当前上下文 | `TransactionService.java` |
| `span.finish()` | 结束 Span | `TransactionService.java` |
| `span.context().traceIdString()` | 获取 traceId 字符串 | `TransactionService.java:currentTraceId()` |
| `span.context().spanIdString()` | 获取 spanId 字符串 | `TransactionService.java:currentSpanId()` |

```java
// TransactionService.java — 交易链路 Demo，演示子 Span 创建
@Service
public class TransactionService {

    @Autowired
    private Tracer tracer;                          // brave.Tracer（Brave 原生）

    // 完整链路：下单 → 扣库存 → 支付
    public Map<String, Object> executeOrder(String userId, String productId, double amount) {
        // ① 下单 — 创建子 Span ORDER
        Span orderSpan = tracer.nextSpan().name("ORDER").start();
        try (Tracer.SpanInScope ws = tracer.withSpanInScope(orderSpan)) {
            sendTrace("ORDER", "SUCCESS", ...);   // 埋点至 Kafka TracingTopic
        } finally {
            orderSpan.finish();
        }

        // ② 扣库存 — 创建子 Span INVENTORY
        Span inventorySpan = tracer.nextSpan().name("INVENTORY").start();
        try (Tracer.SpanInScope ws = tracer.withSpanInScope(inventorySpan)) {
            sendTrace("INVENTORY", "SUCCESS", ...);
        } finally {
            inventorySpan.finish();
        }

        // ③ 支付 — 创建子 Span PAYMENT
        Span paymentSpan = tracer.nextSpan().name("PAYMENT").start();
        try (Tracer.SpanInScope ws = tracer.withSpanInScope(paymentSpan)) {
            sendTrace("PAYMENT", "SUCCESS", ...);
        } finally {
            paymentSpan.finish();
        }
    }

    private String currentTraceId() {
        Span span = tracer.currentSpan();
        return span != null ? span.context().traceIdString() : "N/A";
    }
}
```

#### 如何查看 traceId

## 快速启动

```bash
# 1. 启动中间件（在 192.168.249.129 上）
docker start sentinel-dashboard zipkin    # 或 docker run 新建

# 2. 创建 Kafka Topic（首次）
kafka-topics.sh --create --topic TracingTopic --partitions 3 --replication-factor 1 \
    --bootstrap-server 192.168.249.129:9092

# 3. 启动服务
mvn spring-boot:run -pl chenxu_demo       # :8081 先启
mvn spring-boot:run -pl chenxu-gateway    # :8080 后启

# 4. 验证
curl http://localhost:8080/meter/count                           # Gateway → 服务
curl http://localhost:8080/sentinel/test/flow                    # Sentinel 流控测试
curl -X POST "localhost:8080/order/execute?userId=U001&productId=PROD_001"  # 交易链路
```
