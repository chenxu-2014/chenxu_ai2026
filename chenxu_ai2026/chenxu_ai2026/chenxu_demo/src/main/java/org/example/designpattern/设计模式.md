设计模式主要分三大类，我平时接触比较多的有创建型的**单例**、**建造者**，结构型的**代理**，以及行为型的**策略**和**责任链**

+ **单例模式（Singleton）**：
  - “最常用，比如 Spring 里的 Bean 默认就是单例。我在写工具类或配置类时也会用，确保全局只有一个对象，节省内存。”
  - **进阶补充**：“如果要保证线程安全，我会用静态内部类或枚举方式实现，而不是双重检查锁（DCL），防止指令重排。”
+ **策略模式（Strategy）**：
  - **一定要结合 Sentinel 讲**：“比如我上一家公司做限流降级，针对不同的异常场景（慢调用、异常比例、异常数）定义不同的熔断策略。我通过策略模式封装这些算法，调用方只需要传入 `Rule` 对象，就能动态切换降级策略，避免了大量的 `if-else`。”
  - **或者**：“比如支付渠道（微信/支付宝），定义统一的支付接口，不同渠道实现不同策略。”
+ **责任链模式（Chain of Responsibility）**：
  - **结合 Sentinel 或过滤器讲**：“Sentinel 的底层核心就是责任链，像 `NodeSelectorSlot`、`FlowSlot`、`DegradeSlot` 这些 Slot 串成一条链，依次处理请求的统计、限流、降级。我平时做 Spring Boot 拦截器或参数校验时也用，把校验规则串起来，遇到不满足的直接断掉，**解耦非常彻底**。”
+ **建造者模式（Builder）**：
  - **结合 Kafka 讲**：“就像你刚才问的 Kafka 生产者配置，`Producer` 的配置参数非常多（`acks`、`retries`、`batch.size`），用建造者模式（或者叫链式 `set`）可以清晰地分步构建，避免构造方法参数过多导致混乱。平时用 Lombok 的 `@Builder` 也是同样的道理。”
+ **代理模式（Proxy）**：
  - **结合 AOP 讲**：“Spring AOP 的底层就是动态代理（JDK 或 CGLIB）。我用它做统一日志记录、权限校验和事务管理。代理模式在不入侵原有业务代码的前提下，增强了方法功能，符合**开闭原则**。”

**追问：“单例模式怎么保证线程安全？”**  
答：饿汉式最安全；懒汉式加 `volatile` + `synchronized` 双重检查；或者用静态内部类。

**追问：“代理模式和装饰器模式区别？”**  
答：代理侧重**控制访问**（比如权限），装饰器侧重**增强功能**（比如给咖啡加糖），代理在编译时通常不知道目标对象，装饰器是运行时嵌套的。

**追问：“策略和责任链看起来都有‘链’的感觉，怎么区分？”**  
答：策略是**“选一个”**（一种算法），责任链是**“排着队过”**（一个请求经过多个处理者，直到被处理或遍历完）。
